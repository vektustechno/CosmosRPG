# PLAN.md — CosmoRPG: AI Implementation Plan

> Инструкция для ИИ-агента. Каждый шаг = 1 подсистема (3-8 файлов).
> После каждого шага игра должна запускаться без ошибок.
> Проверка в конце каждого шага обязательна.

---

## Базовые принципы

- **Вид:** top-down, 2D, гексагональная клеточная сетка (6 направлений)
- **Жанр:** Space RPG / Roguelike с элементами 4X и тактики
- **Поток игры:**
  ```
  Сектор (карта звёзд) → Система (планеты + станции + поле боя)
  → Полет по гексам → Бой / Исследование / Сбор
  → Улучшение корабля → Следующий сектор
  ```

## Корабль и его характеристики

### Базовые статы (прокачиваются уровнем + оборудованием)
- **Power** — энергия (расход на стрельбу/движение/щиты)
- **Hull** — прочность корпуса
- **Shield** — щиты (ёмкость + регенерация)
- **Speed** — гексов за ход
- **Turn Rate** — градусов поворота за ход
- **Sensor Range** — радиус обзора
- **Cargo Hold** — грузовой отсек
- **Module Slots** — слоты под модули

### Типы слотов оборудования
| Слот     | Что крепится |
|----------|-------------|
| Weapon   | Оружие (от 2 до 8 в зависимости от класса) |
| Shield   | Генератор щита (1-3) |
| Engine   | Двигатель (1-2) |
| Reactor  | Реактор (1) |
| Armor    | Броня (1-4) |
| Utility  | Утилиты (сканеры, ремкомплекты, хакеры) — 2-4 |
| Special  | Артефакты, экспериментальные модули (1) |

### Типы оружия
- Лазеры (высокая точность, мало энергии)
- Плазма (урон по площади, высокое энергопотребление)
- Ракеты (игнорируют щиты, долгая перезарядка)
- Ионные пушки (отключают системы корабля)
- Рельсотроны (пробивают броню, высокая дальность)
- ЭМИ (сбивает щиты, повреждает электронику)
- Таранный бивень (физический урон в упор)
- Дроны-камикадзе (автономные, запускаются как снаряды)
- Торпеды (медленные, огромный урон)

### Классы кораблей

Игрок начинает на слабом корабле и может купить/получить в награду более мощные. При смене корабля всё съёмное оборудование (модули из слотов) переносится в инвентарь. Часть апгрейдов (уровень, навыки, экипаж) остаётся с игроком.

#### Как получить новый корабль
- **Купить** на верфи станции (требуются Credits + репутация фракции)
- **Награда** за сюжетную миссию (Frigate — после миссии 6, Cruiser — после сектора 4)
- **Дроп с босса** (Dreadnought — только с Dominion Boss, Ancient Carrier — финальный босс)
- **Крафт** из редких компонентов (Battlecruiser)

#### Перенос модулей
- При покупке нового корабля все модули из слотов автоматически снимаются в инвентарь
- Старый корабль можно продать (50% цены) или оставить в гараже на станции
- Если в новом корабле меньше слотов — лишние модули остаются в инвентаре

## Система прокачки (очень развитая)

### А. Уровень корабля (1-99)
- Опыт за убийства и исследования
- +1 к выбранному стату за уровень
- Каждые 10 уровней — перк (+15% к дальности сканера и т.п.)

### Б. Древо навыков (5 ветвей по 30+ узлов)
- **Pilot** — маневренность, уклонение, скорость
- **Gunnery** — урон, перезарядка, точность
- **Engineering** — энергия, ремонт, эффективность щитов
- **Science** — дальность скана, открытие карты, бонус к сбору ресурсов
- **Command** — ауры, напарники, фракционные бонусы

Формат: обычные узлы + редкие "капстоуны" в конце ветки.

### В. Система Crew (экипаж)
- 5 членов (капитан, штурман, инженер, учёный, бортинженер)
- Каждый со своим малым деревом навыков (10 узлов)
- Наём/увольнение на станциях
- Синергии: "Инженер + Учёный = +20% к ремонту щитов"

### Г. Модульная прокачка
- Каждый модуль можно улучшать до +20 уровней
- Тратит ресурсы + кредиты
- На +10 и +20 — открывается дополнительный аффикс

## Система оборудования (очень разнообразная)

### Редкость предметов
```
Broken → Common → Uncommon → Rare → Epic → Legendary → Ancient → Transcendent
```
Каждый уровень редкости добавляет +1 аффикс.

### Аффиксы
- **Оружие:** Fiery (+Fire DoT), Kinetic (+физ.урон), Accurate (+точность), Rapid (+скорострельность), Shield-Breaker, Piercing, Homing, Chain (рикошет)
- **Броня:** Heat-Resistant, Reinforced, Lightweight, Reactive (взрывная реакция)
- **Щиты:** Fast-Recharge, Overcharged, Frequency-Shifting
- **Двигатели:** Warp-Stabilized, Stealth (снижает заметность), Afterburner

### Сеты (set bonuses)
Набор из 2-4 предметов одной серии. Части сета подсвечиваются в инвентаре.

| Сет | Части | 2 части | 3 части | 4 части |
|-----|-------|---------|---------|---------|
| **Void Walker** | shield, engine, utility | +15% shield recharge | +25% + cloak on demand | — |
| **Reaper** | weapon, shield, engine | 5% life steal | 12% life steal, +10% crit | — |
| **Pyro** | weapon, weapon, armor | +25% fire damage | fire immunity, all weapons apply fire | — |
| **Ancient** | weapon, shield, armor, special | +10% all stats | +20%, true damage on crit | +30%, ancient shield (absorbs all types) |

Сетовые предметы:
- Не продаются в обычных магазинах
- Дроп с боссов (25% шанс)
- Крафт по редким рецептам
- Награда за элитные миссии

### Крафт и модификация
- Базовая версия предмета → крафт у NPC на станции
- Модификатор — перековка аффиксов (тратит ресурсы)
- Инкрустация — вставка найденных кристаллов в слоты предмета

## Боевая система

### Основа
- Пошаговая (turn-based) на гексагональной сетке
- 8 AP за ход (базово, меняется от двигателя/реактора/навыков)
- Игрок и враги ходят по очереди

### Расход AP
| Действие | AP |
|----------|----|
| Перемещение на 1 гекс | 1 |
| Поворот на 60° | 1 |
| Поворот на 180° | 3 |
| Выстрел | 2-4 (зависит от оружия) |
| Активация модуля | 1-3 |
| Перекачка энергии между системами | 1 |
| Уклонение (реакция на атаку) | 2 |

### Дуги оружия
- **Forward (90°)** — носовые орудия
- **Broadside L/R (180°)** — бортовые
- **Turret (360°)** — турели, полный круг
- **Rear (90°)** — задние

### Щиты (4 грани)
- Front / Left / Right / Rear — у каждой своё HP
- Перекачка энергии между гранями (1 AP)
- Пробитие грани = урон по корпусу
- Типы щитов: стандартные, направленные, импульсные, аблятивные

### Типы урона
| Тип | Эффект |
|-----|--------|
| Kinetic | +100% к Hull, +50% к Shield |
| Energy | +100% к Shield, +50% к Hull |
| Explosive | AoE 1-3 гекса, +75% к Hull |
| Ion | 0 урона, шанс 40% выключить систему на 2 хода |
| EMP | Сбрасывает щиты, глушит электронику на 1 ход |
| True | Игнорирует щиты и броню (редкое) |

### Статус-эффекты
Горение (DoT 3 хода), ЭМ-глушение, Перегрузка реактора (взрыв через N ходов), Замедление, Захват (бордаж), Отравление, Притяжение (гравитационный луч)

### Системные попадания (Critical Hits)
Шанс 10-30%. Задевает подсистему:
- **Engine** — скорость -50% на 2 хода
- **Weapon Mount** — орудие не стреляет 2 хода
- **Shield Generator** — щиты падают на 1 ход
- **Reactor** — потеря 30% энергии, риск цепной реакции
- **Bridge** — -2 AP в следующем ходу
- **Cargo** — ресурсы вываливаются (можно подобрать)

### Вражеский AI — архитектура поведения

Каждый NPC-противник в начале своего хода выполняет цикл принятия решений:

```
Оценка угрозы → Выбор архетипа поведения → Поиск целей
→ Расчёт действий (с учётом AP) → Выполнение
```

#### Система оценки угрозы
NPC оценивает каждого видимого противника по трём факторам:
- **Dist Threat** — чем ближе, тем выше угроза
- **DPS Threat** — расчётный урон, который противник может нанести за ход
- **Status Threat** — эффекты (например, горящий корабль менее угрожающий, чем невредимый)

Итоговый рейтинг: `threat = dist_weight/distance + dps_weight * estimated_damage + status_modifier`
NPC атакует противника с наивысшим рейтингом, если нет особых указаний.

#### Архитектура AI поведения

Каждый NPC имеет **архетип**, определяющий его поведение в бою. Архетипы реализованы как конечный автомат (FSM) с переходами:

```
         ┌─────────────────────────────┐
         │         IDLE                 │
         │   (движение к цели,        │
         │    сканирование)            │
         └──────────┬──────────────────┘
                    │ враг в радиусе сенсора
         ┌──────────▼──────────────────┐
         │      ENGAGE                 │
         │   (выбор архетипа, целей)   │
         └──────────┬──────────────────┘
                    │
    ┌───────────────┼───────────────┐
    ▼               ▼               ▼
┌─────────┐   ┌─────────┐   ┌─────────┐
│ OFFENSE │   │ DEFENSE │   │ SUPPORT │
└────┬────┘   └────┬────┘   └────┬────┘
     │              │              │
     └──────────────┼──────────────┘
                    │ условие отступления
         ┌──────────▼──────────────────┐
         │      FLEE / RETREAT         │
         │   (HP < порога, перегрев)   │
         └─────────────────────────────┘
```

#### Архетипы поведения (8 штук)

| Архетип | Описание | Условия перехода |
|---------|----------|-----------------|
| **Kamikaze (Drone)** | Летит кратчайшим путём к ближайшему врагу, стреляет всем, что есть, не уклоняется | Всегда, пока жив |
| **Aggressor** | Двигается к цели, держась в диапазоне 3-5 гексов, подставляет слабую сторону щита (провокация) | Если HP > 60% |
| **Brawler** | Идёт в упор (1-2 гекса), использует короткое оружие и таран | Если HP > 50% |
| **Skirmisher** | Держит дистанцию 5-8 гексов, уклоняется при сближении, бьёт с дальней дистанции | Всегда, если есть дальнее оружие |
| **Sniper** | Пытается занять позицию на максимальной дистанции (8-15 гексов), не двигается без нужды | Если есть оружие с range > 6 |
| **Tank** | Встаёт между союзниками и врагом, подставляет сильную сторону щита, использует провокацию/ауру | Если рядом союзники с HP < 50% |
| **Shield Support** | Держится позади танка, восстанавливает щиты союзникам (если есть модуль ремонта) | Если есть ремонтный модуль и союзники с damaged щитами |
| **Juggernaut** | Медленно двигается вперёд, игнорирует уклонение, фокусит одну цель до уничтожения | Всегда (тяжёлые корабли) |

#### Тактические поведения (применяются поверх архетипа)

| Поведение | Описание | Приоритет |
|-----------|----------|-----------|
| **Flank** | Если NPC не единственный — пытается зайти сбоку/сзади, где у цели слабее щиты | Высокий |
| **Focus Fire** | Если 2+ NPC одной группы — все атакуют одну цель целиком | Высокий |
| **Kite** | Если враг быстрее — пятится назад, стреляя, чтобы держать дистанцию | Средний |
| **Shield Balance** | Каждый ход проверяет: если одна грань щита < 30%, перекачивает туда энергию | Средний |
| **Emergency Repair** | Если HP < 30% и есть ремкомплект — тратит AP на восстановление | Высокий |
| **Power Surge** | Если энергия > 80% — временно повышает урон на 20% на 2 хода | Низкий |
| **Fall Back** | Если HP < 20% — пытается отступить за линию союзников | Очень высокий |
| **Overwatch** | Если не двигался в этом ходу — получает бесплатную реактивную атаку по первому вошедшему в радиус | Средний |

#### Формации группы (если врагов > 2)

Когда группа NPC вступает в бой, они распределяют роли:

1. **Лидер** — назначается случайно (или по классу корабля). Лидер получает флаг `is_leader`. Если лидер уничтожен — назначается новый.
2. **Wall** (2 корабля) — встают перед лидером, принимают урон
3. **Flank L / Flank R** — заходят с флангов
4. **Sniper** — отходит на тыловую позицию

Формация перестраивается каждый ход:

```
       [Sniper]
          │
    [Wall] [Leader] [Wall]
       ╱             ╲
  [FlankL]         [FlankR]
```

Если враг атакует фланг — Flank отступает к Wall, Sniper меняет позицию.

#### Как NPC использует оборудование

Каждый NPC создаётся с шаблоном из `enemies.json`, где прописано его оснащение:
- **Тип оружия** и в каком слоте (дуга). NPC знает свои дуги и старается подставить врага под большую часть орудий.
- **Тип щита** — стандартный/направленный (NPC подставляет сильную сторону)
- **Активные модули** — ремонт, ЭМИ, дроны. NPC использует их по таймеру (каждые N ходов) или при условии

Алгоритм выбора оружия:
```
1. Собрать все орудия, способные стрелять по цели (дуга + дальность)
2. Отсортировать по DPS / AP_cost
3. Выбрать лучшее
4. Если AP хватает — выстрелить, иначе выбрать следующее
5. Повторять, пока AP > 0
```
